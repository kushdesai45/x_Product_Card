Introduction
1. Build the results summary UI

Understand Application Requirements

Your challenge is to build out this result summary component using plain HTML and CSS, and get it looking as close to the design as possible.


Your users should be able to:

  • View the optimal layout depending on their device's screen size
  • See hover states for interactive elements


 This is how your application must look upon the initial render:


Implement a hover state that activates when the mouse pointer is over the continue button.

Component Overview

The component displays the following elements:

  • An overall description of the result of a person.
  • The right side of the card shows the marks/100 in different fields.
  • The left side of the card shows:
  • The average of all 4 results on the left-hand side.
  • A brief description of the average result.
  • A continue button on the right side.
  • Attribution section (footer section) with links to Crio and the developer's GitHub profile

NOTE: 

  • The main container must have main as the class name. Use CSS flexbox for creating the layout. The main container's width must be 600px on the Desktop view and 300px on the mobile view.
  • The conclusion text container must have conclusion as the class name. Also, you must use the same text as shown in the screenshot.


  • The Continue button must have the btn as the class name. On hover, the button's background color should change to rgb(28, 26, 138) from its initial state, (refer to provided screenshots). On focus, the button should have outline-style: none.

Verify the Correctness of your Implementation

To test your implementation, run the following commands:


cd assessment
npm install
npx cypress run

All of your test cases will pass if you have implemented the application in the required manner.

If any of the test cases fail:
  • Check for error messages mentioned in the terminal.
  • Make sure that the particular part being tested looks and works as mentioned in the ‘Understanding Application Requirements’ section and is implemented in the required way with all the good practices covered in the session. 
Move to the next milestone if all of your test cases are passing.
